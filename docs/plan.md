# 두바이 쫀득 쿠키(두쫀쿠) 성격 테스트 – 기획 문서 v1

## 1. 한 줄 정의
한국의 유행 디저트 ‘두바이 쫀득 쿠키’를 캐릭터화한 MBTI형 성격 테스트로, 재미와 자기발견을 동시에 제공하고 바이럴 공유를 통해 광고 수익을 극대화하는 웹 서비스.

## 2. 핵심 가치 제안
- 트렌드 감성: 2026 기준 핫한 디저트 밈을 직접 차용
- 몰입형 결과지: 20종 캐릭터 + “뼈 때리는” 성격 묘사
- 공유 유도: 결과 카드/조합 시스템으로 SNS 확산
- AI 하이브리드 분석: 객관식+주관식 통합 분석으로 신뢰도 상승

## 3. 목표 사용자
- 10~30대, SNS 사용 빈도 높음
- 성격 테스트/심리 분석 콘텐츠 공유 경험 있음
- 밈/트렌드 기반 콘텐츠 반응성 높음

## 4. 서비스 흐름 (MVP)
1. 랜딩 → 테스트 시작
2. 객관식 7문항 (빠른 선택)
3. 주관식 3문항 (AI 분석용)
4. 분석 대기 화면 (광고 노출)
5. 결과 페이지 (캐릭터/능력치/한줄평/매칭/공유)
6. 재시도/친구 테스트 유도 (리워드 광고)

## 5. 핵심 기능
- 테스트 진행 UI (10문항)
- AI 분석 (Gemini 1.5 Flash)
- 결과지 생성/저장
- 공유 카드 생성 (SNS용 이미지)
- 광고 삽입 구조 (대기/결과/리워드)

## 6. 수익 구조 (광고 중심)
- 분석 대기 전면 광고
- 결과지 상/하단 배너 광고
- 재시도 시 “두쫀 1코인” → 리워드 광고로 지급
- 공유 후 재시도 보너스 (추가 광고 노출)

## 7. 결과지 구성 (콘텐츠 구조)
- 캐릭터명 & 일러스트
- 뼈 때리는 특징 3~5줄
- 능력치 차트 (눈치력/추진력/진심도 등)
- 환상/환장의 꿀조합 (친구 공유 유도)
- AI 한 줄 평 (주관식 인용 기반)

## 8. 성공 지표 (가설)
- 완료율 60% 이상
- 결과 공유율 25% 이상
- 재시도율 15% 이상
- 광고 CTR 1% 이상

## 9. 리스크 & 대응
- 리스크: 주관식 이탈 → 문항 수 짧게, 자동 저장
- 리스크: 캐릭터 품질 낮음 → “뼈 때리는” 카피 강화
- 리스크: AI 비용 → Flash 모델 + 프롬프트 최적화
- 리스크: 공유 저조 → 매칭/밈형 결과 카드 강조

## 10. 빠른 확장 아이디어
- 시즌 이벤트 캐릭터 (설날/여름 한정 쿠키)
- 결과 카드 템플릿 A/B 테스트
- 커플/우정 조합 기능 강화 (공유율 상승 목적)
